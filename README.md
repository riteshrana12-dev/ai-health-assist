## ✨ Features

### 🤖 AI-Powered Features

| Feature | Description |
|---------|-------------|
| 💬 **AI Health Chatbot** | Symptom checker powered by Gemini AI with real-time emergency detection and conversation history |
| 📄 **Medical Report Analyzer** | Upload PDF/images — AI reads your lab reports and explains results in plain, patient-friendly language |
| 🎯 **Health Score Engine** | AI-calculated composite score (0–100) across BMI, blood pressure, glucose, and lifestyle data |
| 🔮 **Risk Prediction** | Full cardiovascular, diabetes, and metabolic risk assessment with immediate and long-term action plans |
| 📚 **Health Education Library** | 36 topics across 6 categories — AI explains any disease or condition in simple language |
| 💊 **Drug Interaction Checker** | AI analyses your active medications for potentially harmful interactions |
| 🚨 **Emergency Detection** | Pattern-matching engine instantly flags life-threatening symptoms before even calling Gemini |

### 📊 Health Tracking

| Feature | Description |
|---------|-------------|
| 🏠 **Smart Dashboard** | Real-time overview of all vitals with trend arrows and AI-generated insights |
| 📈 **Vitals Logging** | Blood pressure, glucose, BMI, weight, height, heart rate, SpO₂, temperature |
| 📉 **Trend Analytics** | 7/30/60/90-day charts for all metrics with min/max/average summaries |
| 💊 **Medication Tracker** | Full schedule management, dose logging (taken/missed/skipped), adherence tracking |

### 🔐 Platform

| Feature | Description |
|---------|-------------|
| 🔒 **JWT Authentication** | Secure registration, login, token refresh, and protected routes |
| 👤 **Patient Profile** | Full medical history, allergies, lifestyle data, emergency contacts |
| ☁️ **Cloud File Storage** | Cloudinary CDN for medical report PDFs and images |
| 🌙 **Dark / Light Mode** | System-aware theme with persistent user preference |
| 📱 **Fully Responsive** | Optimised for mobile, tablet, and desktop |

## ⚡ Quick Start

### Prerequisites

Make sure you have these installed and ready:

```
✅ Node.js v18+          https://nodejs.org
✅ npm v9+ or yarn
✅ MongoDB Atlas account  https://cloud.mongodb.com     (free)
✅ Gemini API key         https://makersuite.google.com  (free)
✅ Cloudinary account     https://cloudinary.com         (free)
```

---

### 1️⃣ Clone the Repository

```bash
git clone https://github.com/yourusername/ai-health-assist.git
cd ai-health-assist
```

---

### 2️⃣ Setup the Backend

```bash
# Go to server folder
cd server

# Install all dependencies
npm install

# Copy environment template
cp .env.example .env
```

**Edit `server/.env` and fill in your credentials:**

```env
# ── Server ────────────────────────────────────────
PORT=5000
NODE_ENV=development

# ── MongoDB Atlas ─────────────────────────────────
# Get from: cloud.mongodb.com → Connect → Drivers
MONGODB_URI=mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/ai-health-assist

# ── JWT ───────────────────────────────────────────
# Generate: node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
JWT_SECRET=your_64_character_secret_here
JWT_EXPIRES_IN=7d

# ── Google Gemini AI ──────────────────────────────
# Get from: makersuite.google.com/app/apikey (FREE)
GEMINI_API_KEY=AIzaSy...
GEMINI_MODEL=gemini-1.5-flash

# ── Cloudinary ────────────────────────────────────
# Get from: cloudinary.com → Dashboard
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=123456789012345
CLOUDINARY_API_SECRET=your_api_secret

# ── CORS ─────────────────────────────────────────
CLIENT_URL=http://localhost:5173
```

```bash
# Start the backend in development mode
npm run dev

# You should see:
# ✅ MongoDB Connected: cluster0.xxxxx.mongodb.net
# 🚀 AI Health Assist API running on port 5000
```

---

Open a **new terminal window**:


## 📡 API Reference

**Base URL:** `http://localhost:5000/api`

### 🔐 Authentication — `/auth`

| Method | Endpoint | Description | Auth Required |
|--------|----------|-------------|:---:|
| `POST` | `/auth/register` | Register new user | ❌ |
| `POST` | `/auth/login` | Login, returns JWT | ❌ |
| `GET` | `/auth/profile` | Get current user | ✅ |
| `PUT` | `/auth/profile` | Update profile | ✅ |
| `POST` | `/auth/profile/avatar` | Upload avatar image | ✅ |
| `PUT` | `/auth/change-password` | Change password | ✅ |
| `DELETE` | `/auth/account` | Deactivate account | ✅ |

### 💓 Health Data — `/health-data`

## 🗄️ Database Schema

```
MongoDB: ai-health-assist
│
├── users
│   ├── name, email, password (hashed, bcrypt 12 rounds)
│   ├── profile: age, gender, bloodGroup, height, weight
│   ├── medicalHistory[], allergies[], currentMedications[]
│   ├── lifestyle: smoking, alcohol, exercise, sleep
│   ├── emergencyContact: name, relationship, phone
│   ├── settings: theme, notifications, units
│   └── auth: isActive, lastLogin, passwordChangedAt
│
├── healthdatas
│   ├── bloodPressure: systolic, diastolic, pulse, category (auto)
│   ├── glucose: value, unit, mealState, category (auto)
│   ├── bmi: value, category (auto-calculated)
│   ├── vitals: heartRate, oxygenSaturation, temperature
│   ├── labValues: cholesterol, HbA1c, hemoglobin, etc.
│   ├── healthScore: overall (0-100), grade (A-F), breakdown
│   └── riskFlags[]: type, severity, message
│
├── reports
│   ├── fileUrl, cloudinaryPublicId, fileType, fileSize
│   ├── extractedData: rawText, extractionMethod
│   └── aiAnalysis:
│       ├── summary, reportType, confidenceScore
│       ├── keyFindings[]: parameter, value, normalRange, status, explanation
│       ├── riskLevel: low|moderate|high|critical
│       ├── urgentFlags[]: finding, severity, action
│       └── recommendations[], preventiveSuggestions[]
│
└── medications
    ├── name, dosage: amount, unit, form
    ├── schedule: frequency, times[], withMeal
    ├── startDate, endDate, isOngoing
    ├── adherenceLogs[]: scheduledTime, takenAt, status
    └── adherenceRate: 0-100 (auto-calculated)
```

## 🚀 Deployment

### Frontend → Vercel *(Free)*

```bash
# Option A: Vercel CLI
cd client
npm install -g vercel
vercel --prod

# Option B: GitHub Integration
# 1. Push to GitHub
# 2. Import project at vercel.com
# 3. Set Root Directory = client
# 4. Framework = Vite
```

**Required environment variable in Vercel:**
```
VITE_API_URL = https://your-backend-name.onrender.com/api
```

---

### Backend → Render *(Free)*

1. Push your code to **GitHub**
2. Go to [render.com](https://render.com) → **New** → **Web Service**
3. Connect your GitHub repository
4. Configure the service:

```yaml
Root Directory:  server
Build Command:   npm install
Start Command:   node server.js
Node Version:    18
```

5. Add **Environment Variables** (all from `server/.env.example`)
6. Click **Create Web Service** — first deploy takes ~3 minutes

> ⚠️ Free Render instances spin down after 15 min of inactivity.
> The first request after sleep takes ~30 seconds to wake up.

---

### MongoDB Atlas *(Free M0 Cluster)*

```
1. Create account at cloud.mongodb.com
2. Create a FREE M0 cluster (512MB)
3. Database Access → Add Database User
   - Username: aihealth_user
   - Password: (auto-generate strong password)
   - Role: readWriteAnyDatabase
4. Network Access → Add IP Address
   - For Render: Add 0.0.0.0/0 (allow all)
5. Connect → Drivers → Node.js
   - Copy the connection string
   - Replace <password> with your DB user password
   - Add /ai-health-assist before the ?
```

### Cloudinary *(Free 25GB)*

```
1. Create account at cloudinary.com
2. Dashboard → Copy these 3 values:
   - Cloud Name
   - API Key
   - API Secret
3. Set them as CLOUDINARY_* in your .env
```

## 🧪 Testing the API

Test all endpoints with curl or use [Postman](https://postman.com):

```bash
# ── Health check ────────────────────────────────────────
curl http://localhost:5000/api/health

# ── Register ────────────────────────────────────────────
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Test User",
    "email": "test@example.com",
    "password": "test123",
    "age": 28,
    "gender": "male",
    "bloodGroup": "O+"
  }'

# ── Login (save the token!) ──────────────────────────────
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"test123"}'

# ── Log vitals (replace YOUR_JWT_TOKEN) ─────────────────

  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
```
